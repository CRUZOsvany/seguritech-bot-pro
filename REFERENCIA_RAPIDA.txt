🧠 CONCEPTOS CLAVE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

HEXAGONAL ARCHITECTURE
  → Separación en 3 capas: Dominio, Aplicación, Infraestructura
  → El dominio NO conoce detalles técnicos
  → Adaptadores intercambiables

INYECCIÓN DE DEPENDENCIAS
  → Pasar interfaces, no implementaciones
  → Facilita testing y cambios
  → Desacoplamiento total

PUERTOS Y ADAPTADORES
  → Puerto: interfaz (contrato)
  → Adaptador: implementación concreta
  → Cambiar sin tocar el dominio

GESTIÓN DE ESTADO
  → Cada usuario tiene un estado (INITIAL, MENU, MAKING_ORDER, etc)
  → Conversación con contexto
  → Transiciones claras

✅ HACER
  • Mantener el dominio puro (sin dependencias externas)
  • Usar interfaces (puertos)
  • Inyectar dependencias
  • Nombres claros y descriptivos
  • Funciones pequeñas y enfocadas
  • Usar logger en lugar de console.log
  • Separar responsabilidades

❌ EVITAR
  • Mezclar capas
  • Usar "any" types
  • Acoplamiento directo
  • Código duplicado
  • Lógica en adaptadores
  • Cambiar el dominio innecesariamente
  • Nombres genéricos (repo, handler, etc)
